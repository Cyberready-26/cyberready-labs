⚡ Simulation Buttons Explained
▶
Simulate Request
Sends a glowing blue packet through the architecture — Customer Device → Internet → WAF → IDS → Load Balancer → Web Server → Database. Watch the request path light up in real time.
⚠
Simulate Incident
Triggers a mock incident starting at the Database. Watch the monitoring pipeline respond: DB → Metrics → Alert Rules → Alert Manager → Notifications → Runbook. Nodes highlight as the alert propagates.
🔴
Start / Stop Traffic
Activates the Live Metrics Panel and continuously generates request packets through the system. Simulates what real banking traffic looks like. Watch Requests/sec, Latency, and Error Rate update in real time.
⚡
Traffic Spike
Floods the system with a sudden burst of 20 simultaneous requests — simulating a peak event or a potential DDoS situation. Watch metrics spike and alerts fire.
🎓 Training Scenarios (Start Traffic first)
💾
Database Overload
Simulates a slow query problem causing database latency to spike. Watch the alert pipeline activate: DB node highlights → metrics jump → incident logged. Teaches: performance bottleneck detection.
🌊
DDoS Attack
Floods the system with 40 simultaneous packets targeting the WAF. Simulates a Distributed Denial of Service attack. Teaches: how the security layer is the first line of defense under attack.
🧠
Memory Leak
Simulates an application server running out of memory, causing latency and errors. Watch the Web Server node highlight and metrics degrade. Teaches: application-layer failure detection.
🚀
Failed Deployment
Simulates a bad software release causing a surge in error rates. Watch error rate spike and the incident pipeline activate. Teaches: how release engineering failures appear in monitoring data.

🏆 Tips for Best Learning
Follow the sequence first
Go through all 19 steps in order before using the simulation buttons. Understanding the architecture first makes the simulations much more meaningful.
Read the bottom panel
The explanation panel at the bottom of the screen is the most important part. Every node has a detailed description — don't skip it.
Run all 4 scenarios
Each scenario teaches a different failure mode. After completing Layer 4, run all four training scenarios to see how real incidents get detected and escalated.
Combine Simulate Request + Simulate Incident
Start Live Traffic, then immediately trigger Simulate Incident — this most realistically shows how a problem surfaces during normal operations.
